# FinVibe 프론트엔드 개발 규칙

본 문서는 프로젝트의 개발 및 협업을 위한 가이드라인을 담고 있습니다.

## 1. 에이전트 및 응답 가이드라인
- **언어**: 모든 소통은 한국어로 진행하며, 중국어 사용은 엄격히 금지합니다.
- **책임**: 사용자의 요청이 완전히 해결될 때까지 작업을 중단하지 않으며, 작업 완료 후에는 반드시 상태를 확인합니다.

## 2. 코드 작성 규칙
- **경로 임포트**: 상대 경로(`../`) 대신 경로 별칭(`@/`)을 사용합니다.
  - `@/`는 `src/` 디렉토리를 가리킵니다.
  - 옳은 예: `import { Button } from "@/components"`
  - 틀린 예: `import { Button } from "../../components"`

## 3. 커밋 메시지 컨벤션
모든 커밋 메시지는 **한글**로 작성하며 아래 형식을 따릅니다.

### 형식
```text
{action}:메시지

- 상세 변경 내용 1
- 상세 변경 내용 2
- ...
```

### Action 종류
| Action | 설명 |
| :--- | :--- |
| **feat** | 새로운 기능 추가 |
| **refact** | 코드 품질 향상 (리팩토링) |
| **fix** | 버그 수정 |
| **chore** | 빌드 업무 수정, 패키지 매니저 설정, 문서 작업 등 |
